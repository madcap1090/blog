google location tracking

Biking to work this week I was wondering if I had not gotten slower and if my commute was not taking longer than before. Not being a user of Strava or a similar app, I wondered if I could find an answer to that question using my Google location history.

So I downloaded the data, that comes in json format, and had a go at it. You can download yours from your Google account.

Let’s load the data and attach the libraries that are needed.

library("jsonlite")
library("tidyverse")

library("ggplot2")
library("ggmap") # devtools::install_github("dkahle/ggmap")

data <- fromJSON("./data/Location History.json") # extracts a list
locations <- data$locations # and the list contians a dataframe

Let’s plot some data on maps.

First some additional date prep.

df <- df %>% 
  mutate(lat   = latitudeE7/1e7,
         lon   = longitudeE7/1e7,
         time  = strftime(datetime, format = "%H:%M:%S"),
         date  = date(datetime),
         year  = year(datetime),
         month = month(datetime),
         wday  = wday(datetime))

Working with ggmap is pretty straitforward, but you need to obtain an api key from google and enable billing. More info here. For obvious reasons you need to keep your key secret and safe, but additionally you can restrict the calls to your IP.

secrets <- readRDS("~/R/geo/secret/secrets.rds")
key <- secrets$key[1]
register_google(key = key)

belgium <- get_map(location = 'Belgium', zoom = 8)
## Source : https://maps.googleapis.com/maps/api/staticmap?center=Belgium&zoom=8&size=640x640&scale=2&maptype=terrain&language=en-EN&key=xxx
## Source : https://maps.googleapis.com/maps/api/geocode/json?address=Belgium&key=xxx

The map coverage is called the bounding box of the map and it is an attribute of the ggmap object. xxxx

bb <- attr(belgium, "bb")
bb

ggmap(belgium) + geom_point(data = x, aes(x = lon, y = lat), alpha = 0.1, color = "black", size = .5) +

    x = "Longitude",
