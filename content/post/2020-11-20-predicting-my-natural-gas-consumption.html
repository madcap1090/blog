I’ve been recording the gas meter index for almost ten years now.

This is our prime data source:

the gas meter

The data point are from around the end of each month. So this data can be loaded from the xl file.

gas <- readxl::read_excel("./data/20201120/gas.xlsx")
head(gas) %>% 
  reactable(columns = list(
    date = colDef(format = colFormat(date = TRUE)),
    gas_meter = colDef(format = colFormat(separators = TRUE, locales = "en-EN"))
  ), compact = TRUE, width = 250)

Next we will build a time series with an entry for every day. By firstly creating an index, using the tsibble (package)[https://cran.r-project.org/web/packages/tsibble/index.html].

n_days <- as.integer(max(gas$date) - min(gas$date))
days <- ymd(min(gas$date)) + days(0:n_days)
ts_gas <- tsibble(days = days, index = days) %>% 
  rename("date" = "days")

But this is just the evolution of the gas meter. The gas consumption is more interesting. The dplyr, lead function will help us here:

ts_gas <- ts_gas %>% 
   mutate(cons = lead(x = gas_meter, 1, default = NA) - gas_meter )

An plotting it we get:

ggplot(data = ts_gas, aes(x=  date, y = cons))+
  geom_line()+
  theme_minimal()
## Warning: Removed 1 row(s) containing missing values (geom_path).

We can see that the gas consumption forms little plateaus per month, that is of course because we did a linear extrapolation. The chart gives us a clear seasonal pattern. And maybe a downward trend, but we will look at this a bit further down.

Plotting the average consumption per months over the years:

ts_gas_month <- ts_gas %>% 
  as.data.frame() %>% 
  filter(!is.na(date)) %>% 
  mutate(month = as.factor(month(date))) %>% 
  mutate(year = as.factor(year(date))) %>% 
  filter(year!=2010 & year!=2021) %>% 
  group_by(year, month) %>% 
  summarise(cons = mean(cons))
## `summarise()` regrouping output by 'year' (override with `.groups` argument)
ggplot(data = ts_gas_month, aes(x=  month, y = cons, group = year))+
  geom_line(aes(color = year))+
  theme_minimal()

No surprise since we need to heat during the winter months. I used to take holidays in July or August so this is where the minima are situated, since no gas is then used for a couple of weeks.

As you can see the it is between -5 and -7 days that we negative correlation is the highest. I does sometimes feel like there is some inertia in the system, the temperatures are too high for a couple of days when the temperature rises and it takes some time when the weather gets colder, to have nice temperatures inside the house (or it is just because we are only working with average gas consumption date per month).

To conclude let’s take Prophet (Facebook’s time series package) for a spin.

ts <- ts_gas %>% 
  select(date, cons) %>% 
  rename("ds" = "date",
         "y" = "cons") %>% 
  as.data.frame()

m <- prophet(ts)
## Disabling daily seasonality. Run prophet with daily.seasonality=TRUE to override this.
future <- make_future_dataframe(m, periods = (365*2))
forecast <- predict(m, future)

plot(m, forecast)

prophet_plot_components(m, forecast)

Decomposes in trends; monthly, daily and yearly trend. Before 2020 the trend was slightly downwards, due to global warming. Now the trend is upward, due to the confinement. So we do not even enjoy having lower energy bills caused by the climatic crisis.
